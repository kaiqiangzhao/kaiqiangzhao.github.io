python中浮点数的精度

python 中的浮点数

每一种进制都有难以表示的数字。

例如分数1/3

用10进制表示就是一个无限循环小数，0.3333.....

同样使用二进制也很难表示，也是一个无限循环的小数

1/3 * 2 = 2/3 取整数位0

2/3 * 2 = 4/3 取整数位1

1/3 * 2 = 2/3 取整数位0

最后得到的二进制表示的小数就是0.0101010101....

例如十进制数0.1

在平时的编程中，通常使用的是10进制表示的数字，但是计算机却只能使用二进制来存储，因此存在一个进制转换的过程。通常遇到的浮点数精度问题，都是发生在转换过程，就是有些10进制的小数无法使用有限二进制数来表示，这样计算机存储前某几位的二进制数，在此转换为10进制时，就会造成精确不准确的问题。

十进制的0.1 的二进制表示为 0.0001100110011001100110011001100110011001100110011...

eg:

100 * (-19.6) = -1960.0000000000002

floor(100*-19.60) = -1961.0

python 中的round

python中的round并不是绝对的“四舍五入”，而是“四舍六入五成双”

例如round(a.bcd, 2), 即保留两位小数，那么就看小数点后三位：

  1. 如果d大于5，进位
  2. 如果d小于5，舍去
  3. 如果d等于5
  4. d后面没有数据，且c为偶数，那么不进位，保留c
  5. d后面没有数据，且c为奇数，那么进位，c变成c+1
  6. d后面还有非0数字，那就一定要进位，c变成c+1

eg:

round(0.3451, 2) 等于 0，35
round(0.345, 2)  等于 0.34
round(0.335, 2)  等于 0.34
